# フォーム確認ステップ設計（資料請求・無料相談）

2026-07-10 ブレスト確定。送信バックエンド（GAS）は別スコープ — 本specはフロントの確認ステップUIまで。

## 背景・目的

フォーム2種（資料請求 / 無料相談）は現在UIのみ（`onsubmit="return false"`・「※送信機能は準備中」）。
送信前に**入力内容を一度確認させる**フローを作り、誤送信（特に電話番号の打ち間違い）を防ぐ。
ペルソナ＝40代母親・スマホ9割・文章を読まない。

## 確定事項

| 論点 | 決定 |
|---|---|
| スコープ | 確認ステップUIのみ先行。GASバックエンドは後日別セッション |
| 確認UX | **ページ内切替**（入力→確認→完了 の3ステップ。モーダル・別ページは不採用） |
| 送信未接続の間 | 完了画面まで実装するが、**本番は送信ボタンdisabled＋赤字「※送信機能は準備中です」**。`import.meta.env.DEV` でのみ送信ボタン有効（完了画面まで動作確認可。本番ビルドから除去される） |
| URLスラッグ | 資料DL `/document` → **`/request`** にリネーム。無料相談は `/consult` 維持（`contact`はお問い合わせ全般と衝突、`advice`は珍しい形のため不採用） |
| 構成 | ほぼ同一の2フォームを **`components/site/LeadForm.astro`** に一本化。文言・項目は `src/data/forms.ts` に集約（データ駆動） |

## 画面仕様

カード上部にステップ表示「1 入力 → 2 確認 → 3 完了」（現在地を強調）。

1. **入力**: 現行4項目（名前 / 学年 / 電話 / 教室）。ボタン「入力内容を確認する →」。
   HTML5バリデーション＋電話番号形式チェックを通過したら確認へ。
2. **確認**: 4項目を大きめ文字の表で表示。「← 修正する」（フォームは隠すだけなので入力値保持）と
   「この内容で送信する」。切替時はカード先頭へスクロール。
3. **完了**: チェックマーク＋文言。
   - request: 「SMSをお送りしました。届かない場合は番号をご確認のうえ再度お試しください」系
   - consult: 「ご希望の教室からご連絡し、日時を確定します」系
   - 後日GA4のCVイベントはこの到達点に仕込む。

ブラウザ「戻る」との history 連動は今回やらない（シンプル優先）。

## バリデーション

- **selectのrequiredバグ修正**: 「選択してください」の option に `value=""` が無く未選択でも通る → 修正
- **電話番号**: request はSMS送付先のためラベルを「携帯電話番号」にし携帯（070/080/090）のみ許可。
  consult は固定電話も可の一般形式。不正時はフィールド下に赤字エラー
- 全入力に `name` 属性（name / grade / phone / classroom）— GAS送信時にそのまま使う

## 影響ファイル

- 新規: `src/components/site/LeadForm.astro`、`src/data/forms.ts`
- リネーム: `src/pages/document.astro` → `src/pages/request.astro`（consult.astro は LeadForm 利用に書き換え）
- PageHero の英字ラベル（eyebrow）もスラッグに合わせる: 「DOCUMENT」→「REQUEST」（CONSULT は一致済みのまま）
- `/document` リンク張替: Hero / SiteFooter / CtaBand / FixedCta（SP・PC縦の2箇所） / diagnosis
- コメント修正: `src/styles/site.css:84` / `src/lib/path.ts:2`
- サイトは noindex・公開前のためリダイレクト不要
